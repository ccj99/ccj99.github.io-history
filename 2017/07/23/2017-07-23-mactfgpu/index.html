黑苹果 Hackintosh macOS Sierra 10.12.6 安装 CUDA cuDNN Tensorflow-gpu

2017-07-23 Python►Tensorflow 0 Comments

下载安装Xcode

在商店下载Xcode，完成后打开，等待安装组件，直到进入新建项目的页面

注：此步骤目的在于安装xcode command line tool，在后面步骤中已经重复安装低版本，可以先行跳过此步骤，理论没有问题，但我无法重装验证，建议先行跳过，如若因此出错可以再安装

安装HomeBrew及组件

在HomeBrew官网复制命令直接安装即可

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

安装组件

使用如下命令分别安装三个组件

<code>brew install bazel
brew install swig
brew install coreutils
</code>

安装CUDA

在官网下载对应版本的CUDA Driver、CUDA Toolkit，点击安装即可

配置CUDA环境变量

使用命令vi ~/.bash_profile编辑环境变量，将以下三行加在末尾，保存然后退出

<code>export CUDA_HOME="/usr/local/cuda"
export DYLD_LIBRARY_PATH="$DYLD_LIBRARY_PATH:$CUDA_HOME/lib"
export PATH="$CUDA_HOME/bin:$PATH"
</code>

使用命令. ~/.bash_profile重启bash_profile

使用如下命令验证是否安装成功

<code>cd /usr/local/cuda/samples
sudo make -C 1_Utilities/deviceQuery
./bin/x86_64/darwin/release/deviceQuery
</code>

在此处遇到问题

<code>xcode-select: error: tool 'xcodebuild' requires Xcode, but active developer directory '/Library/Developer/CommandLineTools' is a command line tools instance
expr: syntax error
/Developer/NVIDIA/CUDA-8.0/bin/nvcc -ccbin g++ -I../../common/inc  -m64  -Xcompiler -arch -Xcompiler x86_64  -gencode arch=compute_20,code=sm_20 -gencode arch=compute_30,code=sm_30 -gencode arch=compute_35,code=sm_35 -gencode arch=compute_37,code=sm_37 -gencode arch=compute_50,code=sm_50 -gencode arch=compute_52,code=sm_52 -gencode arch=compute_60,code=sm_60 -gencode arch=compute_60,code=compute_60 -o deviceQuery.o -c deviceQuery.cpp
nvcc warning : The 'compute_20', 'sm_20', and 'sm_21' architectures are deprecated, and may be removed in a future release (Use -Wno-deprecated-gpu-targets to suppress warning).
nvcc fatal   : The version ('80100') of the host compiler ('Apple clang') is not supported
make: *** [deviceQuery.o] Error 1
</code>

首先解决第一个问题，需要安装xcode Command line Tool，CUDA不支持cmd tool 8.0以上版本，所以下载Command_Line_Tools_OS_X_10.11_for_Xcode_7.3，此页面需要Apple ID登陆，但不需要注册开发者，登陆后重新打开页面，在列表中选择即可。

下载安装完成后，使用如下命令切换cmd tool

<code>sudo xcode-select --switch /Library/Developer/CommandLineTools
</code>

再使用命令/usr/bin/cc --version查看cmd tool版本，显示如下为安装正确

<code>Apple LLVM version 7.3.0 (clang-703.0.29)
...
</code>

此时再进行编译，通过，用第三条命令执行程序，最后输出

Result = PASS

代表CUDA配置完成，也可以在log中查看显卡型号是否吻合进一步判断。

安装cuDNN5.1

登陆英伟达开发者账号，在链接https://developer.nvidia.com/rdp/cudnn-download下载cuDNN5.1 for CUDA 8.0 Library for OS X，下载后解压并进入该目录，执行如下操作

<code>sudo mv include/cudnn.h /Developer/NVIDIA/CUDA-8.0/include/
sudo mv lib/libcudnn* /Developer/NVIDIA/CUDA-8.0/lib
sudo ln -s /Developer/NVIDIA/CUDA-8.0/lib/libcudnn* /usr/local/cuda/lib/
</code>

安装Anaconda与Python环境

在Anaconda官网下载macOS版本的Ananconda，并安装。

打开Anaconda的图形界面，新建一个环境，命名为tensorflow选择Python 版本为3.6。

新建完成后在环境列表选中该环境，点击小三角，选择open terminal

使用如下命令安装tensorflow 的 GPU 版本

<code>sudo pip install --upgrade https://storage.googleapis.com/tensorflow/mac/gpu/tensorflow_gpu-1.0.1-py3-none-any.whl
</code>

安装成功后使用python命令进入python命令行，输入import tensorflow as tf，提示错误

<code>I tensorflow/stream_executor/dso_loader.cc:135] successfully opened CUDA library libcublas.8.0.dylib locally
I tensorflow/stream_executor/dso_loader.cc:135] successfully opened CUDA library libcudnn.5.dylib locally
I tensorflow/stream_executor/dso_loader.cc:135] successfully opened CUDA library libcufft.8.0.dylib locally
Segmentation fault: 11
</code>

此时打开目录 /usr/local/cuda/lib 将 libcuda.dylib 重命名为 libcuda.1.dylib

重新进入 python命令行，执行import tensorflow as tf，已经正常。继续执行以下两句

<code>hello = tf.constant('Hello, TensorFlow!')
sess = tf.Session()
</code>

输出b'Hello, TensorFlow!'，说明tensorflow安装成功
