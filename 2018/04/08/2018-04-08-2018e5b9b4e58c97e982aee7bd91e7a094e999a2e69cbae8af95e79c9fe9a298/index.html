2018年北邮网研院机试真题

2018-04-08 考研 Comments

2018北邮网研院机试题目与心得

这次网研院的机试，难度出乎意料的低，算是意外情况，所以难度方面不足以作参考，就只谈谈作为一个跨考生应对意外情况的一些技巧吧。

第一题

给出n行商品的单价和数量，单价是小数，求所有物品总花费，保留两位小数。

这个题非常简单，没什么可说的，注意如何输出两位小数就是了，我临场就忘记了，现查的书。所以还是看一眼这种东西，或者带足够的书。

第二题

输出一个数列中V字形三元组的个数，即存在三元组（a，b，c）有 a>b && b<c不要求a,b,c相邻。

这种题首先看数量级，即数列的长度，如果长度不长，直接写三重for循环，遍历出来非常简单。

<code>for(int i = 0; i < n - 2 i ++){
    for(int j = i + 1; j < n - 1; j ++){
        for(int k = j +1; k < n; k ++) {
            if(nums[i] > nums[j] && nums[j] < nums[k]) res++;
        }
    }
}
</code>

侥幸这次机试数量级很小，只有几百，可以直接循环，过了。

第三题

输入四个数字a,b,c,d，若等式a#b#c#d=24成立（#为加减乘除中的任意一种，可以重复出现），那么成该数组满足24点，输出YES，否则输出NO。（计算为实型）

首先，三个运算符各有四种可能，共43 =64种组合，64这个数字不大也不小，手写判定太麻烦也太啰嗦，如果不用手写64种可能，那么考虑三重for循环得到64个式子，for循环index是数字，所以只能做一个数字对运算符的映射，此时的运算符类型为char，若要对其加上数字进行运算，还要进行运算式的识别计算，其中涉及到优先级的判定，除数不为0等，王道机试指南上有源码，不再赘述。相对于手写64个判定条件，运算式的识别更加麻烦，而且程序长，容易出错，而且在实数的条件下，就更加麻烦了。
所以我最终选择手写64个判定条件的方式，但是不同于群里的64if大佬，我就写了一个if。

<code>if(a + b + c + d == 24 || a + b + c - d == 24 || ...)
    printf("Yes");
</code>

提交后又发现错误，原因是我第一次的a,b,c,d都是整形，在除法运算时无法得到正确的结果（例如3/2应该按1.5来计算，而整形的3/2实际结果是1，所以应该将a,b,c,d全部变为double类型。
由于对实型数据输入不熟练，我的处理方法是先输入整形数a1,b1,c1,d1，再转换为double类型，依据就是24点没有用小数玩的。这么说非常牵强和取巧，但是就是想传递一个灵活的思想，当碰到不会的东西的时候，考场上灵活解决很好用。当然再退一步，真的不会也可以写一个test程序跑一下。

第四题

给定n个元件，每个元件i都有一个最佳温度范围[Ri, Ji]，所有原件在最佳温度范围内的性能为y，温度小于Ri性能为x，温度大于Ji性能为z，求使所有原件性能总和最大的温度T。(0<Ri<Ji<109 ,0<n<20000)

这个题目是求最佳状态，貌似是动态规划的问题，但是我动态规划掌握不好，没能找出状态这些东西，所以没做出来，就还是用笨办法试试。
对每个温度进行遍历，分别求出该温度下的性能总和，存起来比较大小，得出最大值和温度。这时候是一个两层循环，外层最多109 次，内层最多20000次，共2*1013 次，数量级庞大，提交代码以后超时，应该想办法精简。
思考可得，显然最佳温度肯定在边界上取得，也就是在Ri或者Ji上，这样就把第一层循环缩小到了40000次，总共8x108 次，提交通过。

<code>int R[n], J[n];
int max = 0;
for(int i = 0; i < n; i ++) {
    int sum = 0;
    for(int j = 0; j < n; j ++) {
        if(R[j] > R[i])  sum += x;
        else if(J[j] < R[i]) sum += z;
        else sum += y;
    }
    if(sum > max) max = sum;
}
</code>
